# GDS Align Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a reusable skill (`nanodevice:gdsalign`) that aligns microscope images to a GDS fabrication template via lithographic marker detection, computes the image→GDS transform, and commits both the warped image and material contours to KLayout.

**Architecture:** 4-script pipeline: (1) extract marker reference positions from Template.gds, (2) detect markers in microscope image via multi-scale template matching, (3) compute similarity transform via exhaustive point-pattern matching + least-squares optimization, (4) warp image + contours and commit to KLayout. Scripts communicate via JSON + .npy files.

**Tech Stack:** Python 3, gdstk, OpenCV (cv2), NumPy, SciPy, argparse CLI

**Spec:** `docs/superpowers/specs/2026-03-13-gdsalign-design.md`

```python
#!/usr/bin/env python
"""Extract the 4 innermost L5/0 marker pairs from a GDS template.

Parses the GDS file, finds L5/0 polygons, selects the 8 closest to the
grid center (775, 775), groups into 4 pairs by proximity, and outputs
pair centroids with bounding boxes.

Usage:
    python extract_markers.py --gds Template.gds --output-dir output/gdsalign/
"""
import argparse
import json
import os
import sys

import gdstk
import numpy as np


def main():
    parser = argparse.ArgumentParser(description="Extract GDS marker pairs")
    parser.add_argument("--gds", required=True, help="Path to Template.gds")
    parser.add_argument("--output-dir", required=True, help="Output directory")
    args = parser.parse_args()

    if not os.path.exists(args.gds):
        print(f"ERROR: GDS file not found: {args.gds}", file=sys.stderr)
        sys.exit(1)

    os.makedirs(args.output_dir, exist_ok=True)

    # Parse GDS and flatten
    lib = gdstk.read_gds(args.gds)
    top_cells = lib.top_level()
    if not top_cells:
        print("ERROR: No top-level cell found", file=sys.stderr)
        sys.exit(1)
    flat = top_cells[0].flatten()

    # Collect L5/0 polygon centroids + bboxes
    markers = []
    for p in flat.polygons:
        if p.layer == 5 and p.datatype == 0:
            bb = p.bounding_box()
            cx = (bb[0][0] + bb[1][0]) / 2
            cy = (bb[0][1] + bb[1][1]) / 2
            markers.append({
                "center": [cx, cy],
                "bbox": [[float(bb[0][0]), float(bb[0][1])],
                         [float(bb[1][0]), float(bb[1][1])]],
            })

    if len(markers) < 8:
        print(f"ERROR: Found only {len(markers)} L5/0 markers, need 8",
              file=sys.stderr)
        sys.exit(1)

    # TODO: Fallback — if caller later reports <3 inner pairs detected,
    # re-run with --ring outer to select the next ring (12 pairs at ~198 um
    # from center, each with 5+7.5 um squares). For now, inner-only.

    # Grid center from all L5 markers
    all_centers = np.array([m["center"] for m in markers])
    grid_center = all_centers.mean(axis=0)

    # Select 8 closest to grid center
    dists = np.sqrt(((all_centers - grid_center) ** 2).sum(axis=1))
    inner_idx = np.argsort(dists)[:8]
    inner = [markers[i] for i in inner_idx]

    # Group into pairs by proximity (nearest neighbor)
    centers = np.array([m["center"] for m in inner])
    used = set()
    pairs = []
    for i in range(len(inner)):
        if i in used:
            continue
        dists_i = np.sqrt(((centers - centers[i]) ** 2).sum(axis=1))
        dists_i[i] = np.inf
        for j in used:
            dists_i[j] = np.inf
        j = int(np.argmin(dists_i))
        used.add(i)
        used.add(j)
        pair_center = (centers[i] + centers[j]) / 2
        pairs.append({
            "center_um": [float(pair_center[0]), float(pair_center[1])],
            "markers": [
                {"bbox": inner[i]["bbox"]},
                {"bbox": inner[j]["bbox"]},
            ],
        })

    # Label pairs by quadrant relative to grid center
    for pair in pairs:
        cx, cy = pair["center_um"]
        if cx > grid_center[0] and cy > grid_center[1]:
            pair["label"] = "NE"
        elif cx < grid_center[0] and cy > grid_center[1]:
            pair["label"] = "NW"
        elif cx < grid_center[0] and cy < grid_center[1]:
            pair["label"] = "SW"
        else:
            pair["label"] = "SE"

    # Sort by label for consistency
    pairs.sort(key=lambda p: p["label"])

    report = {
        "status": "complete",
        "source_gds": os.path.basename(args.gds),
        "layer": "5/0",
        "grid_center_um": [float(grid_center[0]), float(grid_center[1])],
        "pairs": pairs,
    }

    out_path = os.path.join(args.output_dir, "gds_markers.json")
    with open(out_path, "w") as f:
        json.dump(report, f, indent=2)
    print(f"Wrote {out_path} with {len(pairs)} pairs")


if __name__ == "__main__":
    main()
```

- [ ] **Step 3: Implement detect_markers.py**

Create `skills/nanodevice/gdsalign/scripts/detect_markers.py`:

Algorithm summary:
1. Load GDS marker pair geometry from `gds_markers.json`. Take the first pair's two marker bounding boxes.
2. Render a binary template image: two 3×3 um squares (at pixel_size scale, so ~34×34 px each) touching at a corner. The template represents one marker pair (~69×69 px).
3. Convert full_stack image to grayscale (also try inverted for dark-on-light vs light-on-dark markers).
4. For each rotation in [0, 90, 180, 270] and scale in [0.8, 0.9, 1.0, 1.1, 1.2]:
   - Rotate + scale the template using `cv2.getRotationMatrix2D` + `cv2.warpAffine`.
   - Run `cv2.matchTemplate(gray, template, cv2.TM_CCOEFF_NORMED)`.
   - Find peaks above threshold (start at 0.3, lower if too few detections).
5. Non-maximum suppression: merge detections within 1 pair diameter.
6. Map each detection's best rotation to a pair_type (NE/NW/SW/SE) based on the diagonal orientation.
7. Save diagnostics: `01_template.png` (rendered templates at each rotation), `02_correlation_map.png` (best correlation heatmap), `03_detections.png` (full_stack with detected markers circled).
8. Write `image_markers.json`.

Key implementation details:
- The template should be rendered in **both polarities** (white squares on black AND black squares on white) since marker appearance varies with imaging conditions. Run template matching on both, take the better-scoring detections.
- Template rendering: use `cv2.rectangle` on a blank image sized from the bbox geometry.
- NMS: sort by score descending, suppress any detection within `pair_diameter_px` of a higher-scoring one.
- Rotation→pair_type mapping: the NE pair's diagonal goes lower-left→upper-right (45° in image coords), so a template at 0° rotation matches NE, at 90° matches NW, etc. But the actual mapping depends on which pair was used as the base template. Record the rotation and let `align_gds.py` handle the correspondence.

- [ ] **Step 3: Implement align_gds.py**

Create `skills/nanodevice/gdsalign/scripts/align_gds.py`:

Algorithm:
1. Load `gds_markers.json` — extract 4 pair centroids as GDS reference points (in um).
2. Load `image_markers.json` — extract N detected pair centroids (in um, already converted from px using pixel_size).
3. **Exhaustive enumeration**: For each combination of 2 detected points and 2 GDS points (both orderings):
   - Compute the similarity transform from the 2-point correspondence. A similarity transform `[s*cos(θ), -s*sin(θ), tx; s*sin(θ), s*cos(θ), ty]` has 4 DOF. Two point pairs → solve exactly.
   - Apply to all detected points, count inliers (distance to nearest GDS point < tolerance, 1.0 um per spec).
   - Track candidate with most inliers (tie-break: lowest total residual).
4. **Refine**: Using all inlier correspondences from the best candidate, run `scipy.optimize.least_squares` to minimize the total squared residual. The parameter vector is `[θ, s, tx, ty]`.
5. Build the 2×3 affine matrix from optimized `[θ, s, tx, ty]`.
6. Save `gds_warp.npy` and `gds_alignment_report.json`.

Key implementation:

```python
def similarity_from_2_points(src, dst):
    """Compute similarity transform from 2 point pairs.

    src: (2, 2) array of source points
    dst: (2, 2) array of destination points
    Returns: (2, 3) affine matrix
    """
    # Solve [s*cos, -s*sin, tx; s*sin, s*cos, ty]
    # from (x1,y1)→(X1,Y1) and (x2,y2)→(X2,Y2)
    x1, y1 = src[0]
    x2, y2 = src[1]
    X1, Y1 = dst[0]
    X2, Y2 = dst[1]
    # Linear system: A @ [a, b, tx, ty]^T = [X1, Y1, X2, Y2]^T
    # where a = s*cos(θ), b = s*sin(θ)
    A = np.array([
        [x1, -y1, 1, 0],
        [y1,  x1, 0, 1],
        [x2, -y2, 1, 0],
        [y2,  x2, 0, 1],
    ])
    B = np.array([X1, Y1, X2, Y2])
    params = np.linalg.solve(A, B)
    a, b, tx, ty = params
    return np.array([[a, -b, tx], [b, a, ty]])


def refine_similarity(src_pts, dst_pts, M_init):
    """Refine similarity transform via least-squares optimization.

    src_pts: (N, 2) source points
    dst_pts: (N, 2) destination points
    M_init: (2, 3) initial affine matrix
    Returns: optimized (2, 3) affine matrix
    """
    # Extract initial params
    a, b, tx, ty = M_init[0, 0], M_init[1, 0], M_init[0, 2], M_init[1, 2]
    theta = math.atan2(b, a)
    scale = math.sqrt(a**2 + b**2)

    def residuals(params):
        th, s, tx, ty = params
        c, sn = math.cos(th), math.sin(th)
        M = np.array([[s*c, -s*sn, tx], [s*sn, s*c, ty]])
        transformed = (M[:2, :2] @ src_pts.T).T + M[:2, 2]
        return (transformed - dst_pts).ravel()

    result = scipy.optimize.least_squares(
        residuals, [theta, scale, tx, ty], method='lm'
    )
    th, s, tx, ty = result.x
    c, sn = math.cos(th), math.sin(th)
    return np.array([[s*c, -s*sn, tx], [s*sn, s*c, ty]])
```

Create `skills/nanodevice/gdsalign/scripts/commit_gds.py`:

Two modes:
- `--warp-only`: Offline mode — warp image + contours, write output files, no KLayout.
- Default: Full mode — warp + connect to KLayout, load template, add background image, insert polygons.

Algorithm:
1. Load `gds_warp.npy` (2×3 affine, image_um → GDS_um).
2. **Warp image**: Load `full_stack_raw.jpg`, compute output canvas size from GDS template bbox. Use `cv2.warpAffine` with the pixel-domain version of the warp: convert image_um→GDS_um transform to image_px→GDS_px (divide input by pixel_size, multiply output by a chosen output_px_per_um). Save as `full_stack_gds.png`. Also save the image's GDS position metadata (origin_x, origin_y, pixel_size) for the KLayout image skill.
3. **Warp contours**: Load `traces.json`. For each material, transform `contour_um` points using `warp_contour(contour, M)` from `skills/nanodevice/flakedetect/scripts/core.py` (reuse existing tested utility). Save as `traces_gds.json` with new `contour_gds` field. Import via `sys.path.insert(0, os.path.join(os.path.dirname(__file__), "..", "..", "flakedetect", "scripts"))` then `from core import warp_contour`.
4. **KLayout commit** (skip if `--warp-only`):
   a. Connect via `mcp_client.py`, init session.
   b. Load Template.gds: `tool_call("execute_script", code="...")` with pya code to load the GDS.
   c. Add background image via `tool_call("execute_script", code="...")` using pya to create an image overlay with the correct position/scale.
   d. For each material, insert polygon via `tool_call("execute_script", code="...")` using pya `DPolygon` on the appropriate layer (from `layer_map` in traces.json).
   e. Take screenshot for verification.

Key detail for image warp → KLayout background:
